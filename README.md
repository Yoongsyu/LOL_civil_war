# 🎮 League of Legends 내전 관리 및 팀 구성 시스템

이 프로젝트는 롤 내전 플레이어들의 데이터를 관리하고, Riot API를 활용해 실력을 수치화하여 공정한 팀 구성을 도와주는 Streamlit 기반 웹 서비스입니다.

## 🛠 기술 스택
- **Framework**: Streamlit (Python)
- **API**: Riot API (Tier 정보), GitHub API (데이터 저장소)
- **Storage**: GitHub Repository (`data/players.json`, `data/matches.json`)
- **Development**: Google Project IDX + Claude Code

## 📋 핵심 기능 요구사항

### 1. 플레이어 리스트 및 상세 조회 (기능 1, 2, 3)
- **리스트 조회**: 등록된 모든 플레이어를 리스트 형태로 출력합니다.
- **티어 표시**: 플레이어 이름 우측에 Riot API로 가져온 현재 솔랭 티어(예: GOLD I)를 표시합니다.
- **상세 정보**: 이름을 클릭하면 `st.expander` 또는 팝업을 통해 상세 정보(솔랭 티어, 내전 전체 승률, 모스트 포지션, 포지션별 승률/판수)를 대시보드 형태로 보여줍니다.

### 2. 팀 구성 엔진 (기능 4, 5)
- **플레이어 선택**: 리스트에서 내전에 참여할 10명을 체크박스로 선택합니다.
- **[팀 구성하기]**: 
    - 티어와 내전 승률을 합산한 MMR을 기준으로 5:5 팀을 나눕니다.
    - 가장 점수 차가 적은 조합들을 찾고, 그중 **설정된 오차 범위(Tolerance) 내에서 무작위로** 팀을 구성해 매번 결과가 달라지게 합니다.
- **[고정 포지션 팀 구성하기]**:
    - 팀을 나눈 후, 각 팀원들이 가진 포지션별 데이터(승률, 판수)를 분석합니다.
    - 5명의 포지션이 겹치지 않으면서 **팀 전체의 포지션 숙련도 합이 최대**가 되도록 5명에게 (탑, 정글, 미드, 원딜, 서포터) 역할을 자동 배정합니다.

### 3. 플레이어 등록 (기능 6)
- **Riot API 연동**: "닉네임#태그"를 입력받아 Riot API를 호출합니다.
- **자동 저장**: 해당 유저의 PUUID와 현재 티어 정보를 가져와 초기 MMR을 계산하고 `players.json`에 신규 등록합니다.

### 4. 전적 관리 및 관리자 메뉴 (기능 7, 8)
- **관리자 모드**: 사이드바 혹은 별도 탭에서 관리자 비밀번호를 입력하여 인증합니다.
- **전적 입력/수정**: 
    - 내전 종료 후 (승리팀, 패배팀, 각 플레이어의 포지션)을 선택하여 등록합니다.
    - 등록 시 `players.json`의 승률, 판수, 포지션별 통계가 즉시 업데이트되며 GitHub에 커밋됩니다.
    - 잘못 입력된 이전 경기를 수정/삭제하는 기능을 포함합니다.

💡 개발 지시 사항 (Claude Code용)
초기 설정: PyGithub를 활용하여 github_utils.py를 먼저 작성하고, JSON 데이터를 읽고 쓰는 기초를 다져줘.
API 연동: riot_api.py에서 Riot ID로 티어와 PUUID를 가져오는 함수를 작성해줘.
UI 구현: app.py에서 플레이어 리스트(티어 포함)와 상세 정보 확인 기능을 구현해줘.
알고리즘: balancer.py에서 10명을 선택했을 때 팀 점수 차이가 최소가 되는 조합 중 랜덤하게 하나를 뽑는 balance_teams 함수를 작성해줘.
관리자 기능: 관리자 비번 인증 후 경기 결과를 입력하면 JSON 파일의 stats 데이터가 업데이트되는 로직을 완성해줘.
